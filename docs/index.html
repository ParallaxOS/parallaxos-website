ParallaxOS · Documentation

Welcome to ParallaxOS.

ParallaxOS is the sovereign compliance operating system for Australian rail subcontractors. This documentation covers everything from a five-minute getting-started guide to the architecture of Edge Mode and the public REST API.

New to ParallaxOS? Start with the Getting Started guide for a 30-minute onboarding walkthrough, or read the Three Portals concept page to understand how Admin, Supervisor, and Worker views fit together.

What is ParallaxOS?

ParallaxOS replaces the spreadsheets, paper forms, and email chains of Australian rail compliance with one purpose-built platform. It runs across three portals — Admin (desktop), Supervisor (tablet), and Worker (mobile) — wired into every major Australian rail network and to Avetta, RIW, Xero, and MYOB.

The core promise: replace two to five days of manual compliance admin every month with a system that generates the Avetta Monthly Pack in three seconds and pushes it directly to the API.

Conventions used in these docs

Where you see a pill like CLEARED, that's a real status indicator from the platform. Status pills are colour-mapped throughout the product:

  • Cleared — worker is compliant and rosterable
  • Blocked — worker cannot be assigned (FAID, expired cert, D&A positive)
  • Warning — action required (expiring within 30 days)
  • Info — informational only
  • AI — generated or actioned by ParallaxOS AI
  • Asset — fleet plant or equipment context

Code samples use JetBrains Mono and follow the platform's actual conventions. API examples are curl by default, with JavaScript SDK snippets where relevant.
